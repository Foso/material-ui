Grid

Material Design 响应式布局的栅格可适应屏幕大小和方向，确保布局之间的一致性。

Grid: 栅格(Grid)组件能确保不同布局间的视觉一致性，同时在众多不同设计中保持灵活性。 Material Design 的响应式 UI 是基于12列的栅格布局。

它是如何工作的

栅格系统使用 Grid 组件实现：

  • 为了达到高度的灵活性，它运用了用 CSS 的 Flexible Box 模块 。
  • 它有两种类型的布局： containers ， items。
  • 项目宽度以百分比设置，因此它们总是相对于其父元素是流动的和变换大小的。
  • 子项自带 padding 来和其他元素之间产生间距。
  • 有五个网格断点：xs，sm，md，lg 和 xl。

如果你对 flexbox 不熟悉，我们建议你阅读：CSS-Tricks flexbox 手册。

Spacing（间距）

响应式栅格侧重于一致的间距宽度，而不是列宽。 Material design 外边距和列遵循** 8px **的方块形基线栅格。 spacing 属性设置为一个在0和10之间的整数，且并包括0和10。 默认情况下，两个网格项之间的间距遵循这样的线性函数： output(spacing) = spacing * 8px，例如 spacing={2} 会创建一个 16px 的宽间距。

通过使用主题，该变换函数的输出是可定制的。

Fluid grids（流式格网）

流体网格使用的列可以缩放和调整内容的大小。 流体网格的布局可以通过使用断点来确定布局是否需要显式更新。

局限性

负边距

当我们使用负边距来实现项目之间的间距的时候，会有一个限制。 如果负边距超出<body>元素，则会出现水平滚动。 这里有3种可用的解决方案：

  1. 不使用 spacing 的特性并且设置成默认的spacing={0}
  2. 将填充应用于父级元素，并且将至少一半的间距值赋予子级元素：
  <body>
    <div style={{ padding: 20 }}>
      <Grid container spacing={5}>
        //...
      </Grid>
    </div>
  </body>
  1. 将父元素设置为 overflow-x: hidden;。

white-space: nowrap;

弹性布局元素的默认属性值为 min-width：auto。 当子元素使用 white-space: nowrap;时会出现布局冲突。 您可以从以下内容看到这个问题：

<Grid item xs>
  <Typography noWrap>

为了使项留在容器内，您需要设置 min-width: 0。 实际上，您也可以通过设置 zeroMinWidth 属性解决问题：

<Grid item xs zeroMinWidth>
  <Typography noWrap>

direction: column | column-reverse

虽然 Grid 组件有 direction 属性，所以组件能用这些的值：row，row-reverse，column，和column-reverse，但是有些功能是不被 column 和column-reverse 容器支持的。 一些定义了组件栅格数量的属性会用于一个特定的断点：（ xs，sm，md，lg 以及 xl) ， 而这这些属性主要来控制宽度，并且不会再 column 和 column-reverse 内产生对高度相同的效果。 如过在 column 或者 column-reverse 容器内使用这些属性,，将会对 Grid 元素产生意想不到的效果。
